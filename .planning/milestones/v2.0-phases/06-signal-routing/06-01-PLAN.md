<objective>
Build the inverted index that maps graph elements to the frames they affect, enabling O(affected) event routing.

Purpose: When an event arrives, Krabnet needs to know which frames are affected without scanning all frames. The inverted index maintains posting lists mapping node IDs and (source, edge_type) pairs to frame ID sets, enabling O(affected) lookup.

Output: `src/routing.rs` with InvertedIndex, register/unregister, affected_frames query, and comprehensive tests.
</objective>

<tasks>

<task id="1" name="Create inverted index for signal routing" commit_prefix="feat(06-01)">
Create `src/routing.rs`:

**InvertedIndex struct:**
- `node_to_frames: HashMap<NodeId, HashSet<u64>>` — maps node IDs to frame IDs containing that node
- `edge_key_to_frames: HashMap<(NodeId, TypeId), HashSet<u64>>` — maps (source_node, edge_type) pairs to frame IDs

**Operations:**
- `new() -> Self`
- `register_frame(&mut self, frame_id: u64, node_ids: &[NodeId], edge_keys: &[(NodeId, TypeId)])` — add frame_id to all relevant posting lists for the given nodes and edge keys
- `unregister_frame(&mut self, frame_id: u64, node_ids: &[NodeId], edge_keys: &[(NodeId, TypeId)])` — remove frame_id from all relevant posting lists, clean up empty sets
- `affected_frames(&self, event: &Event) -> HashSet<u64>` — return deduplicated union of all frame IDs affected by event:
  - `NodeAdded { node_id, .. }` → lookup node_id
  - `NodeRemoved { node_id }` → lookup node_id
  - `EdgeAdded { source, target, type_id, .. }` → union of lookup(source), lookup(target), lookup((source, type_id))
  - `EdgeRemoved { source, target, .. }` → union of lookup(source), lookup(target)
  - `PropertyChanged { node_id, .. }` → lookup node_id
- `frame_count(&self) -> usize` — count of unique frame IDs across all posting lists

**src/lib.rs:** Add `pub mod routing;` and `pub use routing::InvertedIndex;`

**Verification:** `cargo build && cargo clippy` — zero errors, zero warnings
</task>

<task id="2" name="Add comprehensive routing tests" commit_prefix="feat(06-01)">
Add tests in `src/routing.rs`:

- `register_and_lookup_by_node` — register frame with nodes, lookup each node returns frame
- `affected_frames_edge_added` — EdgeAdded event hits source, target, and edge_key posting lists
- `affected_frames_deduplicated` — shared node in multiple frames returns all frame IDs
- `shared_node_fan_out` — node in 3 frames, event on that node returns all 3

- `unregister_cleans_empty_sets` — empty posting lists are removed
- `affected_frames_property_changed` — PropertyChanged on node returns frames containing that node
- `affected_frames_node_removed` — NodeRemoved returns frames containing that node
- `empty_index_returns_empty` — no registrations, any event returns empty set
