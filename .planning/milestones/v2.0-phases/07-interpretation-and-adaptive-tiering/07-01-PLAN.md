<objective>
Implement the two-tier interpretation system and adaptive frame tiering.

Purpose: Tier 1 is a fast O(1) binary check — did the frame's net delta change? If yes, escalate to Tier 2 for structural path analysis. The tiering system scores frames by access pattern (query frequency, mutation rate, recency) and classifies them as Hot/Warm/Cold.

Output: `src/interpret.rs` (Tier 1 binary check, Tier 2 structural analysis) and `src/tiering.rs` (priority scoring, tier recommendation) with tests.
</objective>

<tasks>

<task id="1" name="Create interpretation and tiering modules" commit_prefix="feat(07-01)">
Create two modules:

**src/interpret.rs:**
- `tier1_check(previous_net_delta: i64, current_net_delta: i64) -> bool` — returns true if changed, false if same
- `HopAnalysis` struct: `hop_index: usize`, `completed: usize` (paths that completed this hop), `broken: usize` (paths that broke at this hop)
- `tier2_analysis(frame: &Frame, graph: &Graph, epoch: Epoch) -> Vec<HopAnalysis>` — for each hop in the frame's pattern, count how many current-state paths pass through that hop level vs break. A path "completes" a hop if it has nodes at that hop's position. A path is "broken" at hop N if it has fewer than N+1 nodes after the anchor (i.e., path is too short).
  - Actually simpler: get snapshot paths, for each hop index, count paths that have length >= hop_index + 2 (completed) vs paths that have length == hop_index + 1 (broken at this hop). Return analysis per hop.
- Doc comments on all public items

**src/tiering.rs:**
- `TierConfig` struct: `query_weight: f64` (default 0.4), `mutation_weight: f64` (default 0.3), `recency_weight: f64` (default 0.3), `recency_half_life: f64` (default 100.0 epochs)
- `impl Default for TierConfig` with above defaults
- `priority_score(query_count: u64, mutation_count: u64, epochs_since_last: u64, config: &TierConfig) -> f64` — normalized 0.0-1.0 score:
  - query_component = (query_count as f64).ln_1p() / 10.0 (capped at 1.0)
  - mutation_component = (mutation_count as f64).ln_1p() / 10.0 (capped at 1.0)
  - recency_component = (-epochs_since_last as f64 / config.recency_half_life).exp()
  - raw = query_component * config.query_weight + mutation_component * config.mutation_weight + recency_component * config.recency_weight
  - return raw.clamp(0.0, 1.0)
- `recommend_tier(score: f64) -> FrameTier` — Hot if > 0.7, Cold if < 0.2, else Warm
- Doc comments on all public items

**src/lib.rs:** Add `pub mod interpret;` and `pub mod tiering;`
