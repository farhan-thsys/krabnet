<objective>
Implement background compaction worker and multi-threaded frame evaluation to make the engine survive concurrent load.

Purpose: The v1.0 engine performs compaction synchronously and evaluates frames sequentially on the main thread. Under load, tuple counts grow unbounded and frame evaluation becomes a bottleneck. Background compaction prevents memory exhaustion; parallel frame evaluation utilizes multiple cores.

Output: New `src/compaction.rs` module with CompactionWorker, updated `src/engine.rs` with Arc<RwLock<Frame>> wrapping and thread-pool evaluation, updated Cargo.toml with parking_lot dependency.
</objective>

<task type="auto">
  <name>Task 1: Create CompactionWorker with background thread and double-buffer compaction</name>
  <files>src/compaction.rs, src/lib.rs, Cargo.toml</files>
  <action>
1. Add `parking_lot = "0.12"` to `[dependencies]` in Cargo.toml.

2. Create `src/compaction.rs` with:

   - `CompactionStats` struct: `compactions_completed: u64`, `tuples_before: u64`, `tuples_after: u64`, `total_compaction_time_us: u64`. Derive Debug, Clone, Default.

   - `CompactionRequest` struct: holds an `Arc<parking_lot::RwLock<Frame>>` (the frame to compact) and an `Epoch` frontier. The compaction worker uses double-buffering: acquire read lock, clone the DiffCollection, release read lock, compact the clone off-lock, then acquire write lock and swap the compacted collection back. The Mutex (write lock) is held ONLY during the swap, not during the full compaction.

   - `CompactionWorker` struct:
     - `sender: crossbeam::channel::Sender<CompactionRequest>` (for sending requests to the worker)
     - `stats: Arc<parking_lot::Mutex<CompactionStats>>` (shared stats, parking_lot::Mutex for low-overhead reads)
     - `threshold: usize` (default: 10_000, configurable)
     - `handle: Option<std::thread::JoinHandle<()>>` (the background thread)

   - `CompactionWorker::new(threshold: usize) -> Self`:
     - Create a crossbeam unbounded channel
     - Spawn a `std::thread::Builder::new().name("compaction-worker")` thread
     - The thread loop: `while let Ok(request) = receiver.recv()` {
       1. Acquire READ lock on the Arc<RwLock<Frame>>, clone the DiffCollection, read tuple count before, release read lock.
       2. Compact the cloned DiffCollection off-lock (call compact(frontier) on the clone). This is the expensive step and does NOT hold any lock.
       3. Acquire WRITE lock on the Arc<RwLock<Frame>>, swap the compacted DiffCollection back into the Frame (via Frame::swap_diff_collection), read tuple count after, release write lock. The write lock is held ONLY during the swap.
       4. Update stats.
     }
     - Track timing via `std::time::Instant`

   - `CompactionWorker::request_compaction(&self, frame: Arc<parking_lot::RwLock<Frame>>, frontier: Epoch)`:
     - Sends a CompactionRequest through the channel (non-blocking, use `send` which blocks only if channel is full -- use unbounded so it never blocks)

   - `CompactionWorker::should_compact(&self, tuple_count: usize) -> bool`:
     - Returns `tuple_count >= self.threshold`

   - `CompactionWorker::stats(&self) -> CompactionStats`:
     - Lock stats mutex (parking_lot), clone and return

   - `CompactionWorker::shutdown(self)`:
     - Drop the sender (causes recv to return Err, breaking the loop)
     - Join the thread handle

   - Implement `Drop` for CompactionWorker: drop sender, join handle (if not already taken).

3. Add `pub mod compaction;` and `pub use compaction::CompactionWorker;` to `src/lib.rs`.

4. Add co-located `#[cfg(test)] mod tests` in compaction.rs:
   - `test_compaction_worker_compacts_on_request`: Create a Frame with 100 assert+retract pairs (using frame.ingest() or direct DiffCollection manipulation), wrap in Arc<parking_lot::RwLock<Frame>>, send compaction request, sleep briefly, verify tuples reduced by reading through the RwLock. Verify readers are not blocked during compaction (acquire a read lock concurrently while compaction is in progress -- the double-buffer pattern ensures the write lock is held only during the swap).
   - `test_compaction_stats_tracking`: Send a compaction request, wait for completion, verify stats.compactions_completed increments and tuples_before/after are correct.
   - `test_should_compact_threshold`: Verify threshold logic (below = false, at/above = true).

Note: CompactionRequest holds Arc<parking_lot::RwLock<Frame>> + Epoch. The worker uses double-buffering per COMPACT-02: read lock to clone, compact the clone off-lock, write lock only to swap back. This ensures readers are NOT blocked during the expensive compaction step. Use parking_lot::Mutex for stats (lightweight, low-overhead reads).

  <name>Task 2: Integrate compaction worker and multi-threaded frame evaluation into Engine</name>
  <files>src/engine.rs, src/frame.rs</files>
  <action>
1. Modify `src/frame.rs`:
   - Add `pub fn swap_diff_collection(&mut self, compacted: DiffCollection)` method: replaces the Frame's internal DiffCollection with the provided compacted one. This is called by the compaction worker while holding a write lock, completing the atomic swap step of the double-buffer pattern (COMPACT-02). The write lock is held only for this swap, not during the actual compaction work.
   - Add `pub fn clone_diff_collection(&self) -> DiffCollection` method: returns a clone of the Frame's internal DiffCollection. Called by the compaction worker while holding a read lock, before releasing the lock to compact off-lock.
   - Add `pub fn tuple_count(&self) -> usize` if not already present (needed for the threshold check in engine.rs and for CompactionStats tracking).

2. Modify `src/engine.rs`:
   - Change `frames: HashMap<u64, Frame>` to `frames: HashMap<u64, Arc<parking_lot::RwLock<Frame>>>`.
   - Add `compaction_worker: Option<CompactionWorker>` field.
   - Add `Engine::with_compaction(ring_buffer_capacity: usize, compaction_threshold: usize) -> Self` constructor that creates the CompactionWorker.
   - Update `Engine::new()` to NOT create a CompactionWorker (backward compatible).
   - Update ALL frame access patterns throughout Engine to use `.read()` or `.write()` locks:
     - `ingest()`: After index lookup, use `std::thread::scope` to fan out frame evaluation. For each affected frame_id, spawn a scoped thread that acquires a write lock on the frame and performs the tier1_check. The inverted index lookup stays on the main thread (EVAL-03). After evaluation, check if any frame's tuple_count exceeds the compaction threshold and send a compaction request if so.
     - `register_frame()`: Create frame, wrap in Arc<RwLock<>>, insert.
     - `query_frame()`: Acquire write lock (query increments count), call query, collect results.
     - `snapshot_frame()`: Acquire read lock, call snapshot.
     - `compact_all()`: Acquire write lock on each frame, call compact.
     - `stats()`: Acquire read lock on each frame.
     - `extract_node_ids_from_frame()`: Takes `&Arc<RwLock<Frame>>`, acquires write lock.
   - For the fan-out evaluation in `ingest()`:
     - Collect affected frame_ids from inverted index (main thread, EVAL-03)
     - Use `std::thread::scope(|s| { for frame_id in affected { s.spawn(|| { ... }) } })` to evaluate frames in parallel (EVAL-01)
     - Each spawned thread: acquire write lock, run tier1_check, update previous_deltas (need thread-safe access -- use a local collection and merge after scope)
   - For compaction integration in ingest():
     - After frame evaluations, if compaction_worker is Some, iterate frames, check tuple_count against threshold, and request compaction for those exceeding it
     - Clone the Arc<RwLock<Frame>> and send it to the compaction worker along with the frontier epoch. The worker uses double-buffering: (1) read lock + clone DiffCollection, (2) compact clone off-lock, (3) write lock + swap back. Readers are blocked only during the swap, not during compaction.
     - CompactionRequest already holds `Arc<parking_lot::RwLock<Frame>>` + `Epoch` (defined in Task 1) -- no redefinition needed.
   - Add `compaction_stats(&self) -> Option<CompactionStats>` method.
   - Embryonic auto-promotion: wrap new frames in Arc<RwLock<>>.
   - Handle `previous_deltas` thread-safety: Use `Arc<parking_lot::RwLock<HashMap<u64, i64>>>` for previous_deltas so scoped threads can update it. Or simpler: collect (frame_id, new_delta) pairs from scoped threads and merge on main thread after scope returns.

3. Ensure all existing engine tests still compile and pass. The public API (ingest, register_frame, query_frame, etc.) should remain the same. Internal locking is transparent to callers.

Important: Do NOT change the `&mut self` signature of `ingest()` -- the engine itself is still single-writer. The parallelism is within the ingest call (fan-out to frame evaluators), not across ingest calls.
