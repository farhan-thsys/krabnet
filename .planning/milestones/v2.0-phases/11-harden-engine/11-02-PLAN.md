<objective>
Implement mutation coalescing, fan-out limits, and frame prioritizer hysteresis to prevent pathological behavior under load.

Purpose: Without coalescing, a burst of mutations to the same node triggers redundant frame evaluations. Without fan-out limits, a super-node event can trigger thousands of frame evaluations, stalling the pipeline. Without hysteresis, oscillating scores cause frames to thrash between Hot and Cold tiers. These three mechanisms harden the engine for realistic workloads.

Output: New `src/coalescer.rs` with MutationCoalescer, new `src/fanout.rs` with FanOutLimiter and DeferredEvalQueue, updated `src/tiering.rs` with HysteresisState.
</objective>

<task type="auto">
  <name>Task 1: Create MutationCoalescer with epoch-window deduplication</name>
  <files>src/coalescer.rs, src/lib.rs</files>
  <action>
1. Create `src/coalescer.rs` with:

   - `CoalescedEntry` struct: `node_id: NodeId`, `latest_event: Event`, `epoch_start: Epoch`, `epoch_end: Epoch`. Derive Debug, Clone.

   - `CoalescedBatch` struct: `entries: Vec<CoalescedEntry>`. Derive Debug, Clone. This is the output containing deduplicated (node_id, latest_event, epoch_range) tuples (COALESCE-03).

   - `MutationCoalescer` struct:
     - `window_size: u64` (default: 16 epochs, configurable per COALESCE-01)
     - `pending: HashMap<NodeId, CoalescedEntry>` (accumulates events per node within the current window)
     - `window_start: Option<Epoch>` (start epoch of the current window)

   - `MutationCoalescer::new(window_size: u64) -> Self`

   - `MutationCoalescer::push(&mut self, node_id: NodeId, event: Event, epoch: Epoch) -> Option<CoalescedBatch>`:
     - If window_start is None, set it to epoch (first event starts the window).
     - If epoch >= window_start + window_size, flush the current window (return a CoalescedBatch), then start a new window with this event.
     - Otherwise, upsert into pending: if node_id already exists, update latest_event and epoch_end. If not, insert new entry. Same-node mutations within window collapse to single entry (COALESCE-02).
     - Return None if window hasn't elapsed.

   - `MutationCoalescer::flush(&mut self) -> CoalescedBatch`:
     - Drain all pending entries into a CoalescedBatch.
     - Reset window_start to None.

   - `MutationCoalescer::pending_count(&self) -> usize`: Returns number of distinct nodes pending.

   - Helper to extract NodeId from Event:
     - `fn event_node_id(event: &Event) -> Option<NodeId>`: Returns the primary node_id affected by the event (NodeAdded/NodeRemoved/PropertyChanged -> node_id, EdgeAdded/EdgeRemoved -> source).

2. Add `pub mod coalescer;` and `pub use coalescer::MutationCoalescer;` to `src/lib.rs`.

3. Add co-located `#[cfg(test)] mod tests`:
   - `test_same_node_coalescing`: Push 100 mutations to the same NodeId within one window, flush, verify batch has exactly 1 entry with correct epoch_range.
   - `test_different_nodes_preserved`: Push mutations to 10 different NodeIds within one window, flush, verify batch has 10 entries (COALESCE-02 inverted: different-node mutations preserved).
   - `test_window_auto_flush`: Push events spanning past window_size, verify auto-flush returns a batch.
   - `test_default_window_size`: Verify default window_size is 16.
  </action>
  <verify>
    <automated>cargo test --lib coalescer -- --nocapture</automated>
    <manual>Verify MutationCoalescer deduplicates same-node mutations and preserves different-node mutations</manual>
  </verify>
  <done>MutationCoalescer accumulates events within configurable epoch window (default 16), same-node mutations collapse to single trigger, different-node mutations all produce separate triggers, CoalescedBatch contains deduplicated entries</done>
</task>

<task type="auto">
  <name>Task 2: Create FanOutLimiter with DeferredEvalQueue and add HysteresisState to tiering</name>
  <files>src/fanout.rs, src/tiering.rs, src/lib.rs</files>
  <action>
1. Create `src/fanout.rs` with:

   - `DeferredEvalEntry` struct: `frame_id: u64`, `priority_score: f64`. Derive Debug, Clone.

   - `DeferredEvalQueue` struct:
     - `queue: Vec<DeferredEvalEntry>` (sorted by priority_score descending)
   - Methods:
     - `new() -> Self`
     - `push(&mut self, frame_id: u64, priority_score: f64)`: Insert and maintain sorted order (use binary_search_by for insertion point, sorted descending by priority_score).
     - `pop_batch(&mut self, count: usize) -> Vec<u64>`: Pop up to `count` highest-priority frame_ids.
     - `len(&self) -> usize`
     - `is_empty(&self) -> bool`

   - `FanOutLimiter` struct:
     - `max_fanout: usize` (default: 1000, configurable per FANOUT-01)
     - `deferred: DeferredEvalQueue`
   - Methods:
     - `new(max_fanout: usize) -> Self`
     - `limit(&mut self, affected_frame_ids: Vec<(u64, f64)>) -> (Vec<u64>, usize)`:
       Takes a list of (frame_id, priority_score) pairs. If len <= max_fanout, returns all as immediate, 0 deferred. If len > max_fanout, sorts by priority_score descending, takes top max_fanout as immediate, pushes remainder into DeferredEvalQueue. Returns (immediate_frame_ids, count_deferred) (FANOUT-02).
     - `drain_deferred(&mut self, count: usize) -> Vec<u64>`: Pop up to `count` deferred frames for later processing.
     - `deferred_count(&self) -> usize`

2. Add `pub mod fanout;` and `pub use fanout::FanOutLimiter;` to `src/lib.rs`.

3. Update `src/tiering.rs` -- add HysteresisState:

   - `HysteresisState` struct:
     - `consecutive_below_cold: u32` (how many consecutive windows score was below cold threshold)
     - `consecutive_above_hot: u32` (how many consecutive windows score was above hot threshold)
     - `required_consecutive: u32` (default: 5, configurable per HYST-02/HYST-03)
   - Methods:
     - `new(required_consecutive: u32) -> Self`
     - `update(&mut self, score: f64, current_tier: FrameTier) -> FrameTier`:
       - If score > 0.7: increment consecutive_above_hot, reset consecutive_below_cold. If consecutive_above_hot >= required_consecutive AND current_tier != Hot, return Hot. Else return current_tier.
       - If score < 0.2: increment consecutive_below_cold, reset consecutive_above_hot. If consecutive_below_cold >= required_consecutive AND current_tier != Cold, return Cold. Else return current_tier.
       - If 0.2 <= score <= 0.7: reset BOTH counters. Return Warm (Warm is the "safe" middle state -- oscillating scores stay Warm per HYST-01).

4. Add co-located tests in fanout.rs:
   - `test_under_max_fanout`: 500 frames, max_fanout=1000, all returned as immediate.
   - `test_over_max_fanout`: 2000 frames, max_fanout=1000, top 1000 by priority returned immediate, 1000 deferred.
   - `test_deferred_queue_priority_order`: Verify deferred frames are popped in priority order.
   - `test_drain_deferred`: Push excess, drain_deferred returns correct batch.

5. Add co-located tests in tiering.rs:
   - `test_hysteresis_prevents_thrashing`: Oscillate score between 0.1 and 0.8 each window for 20 windows. With required_consecutive=5, tier should stay Warm (never reaches 5 consecutive below or above). This directly validates HYST-01.
   - `test_hysteresis_promotes_after_consecutive`: Score above 0.8 for 5 consecutive windows, verify promotion to Hot.
   - `test_hysteresis_demotes_after_consecutive`: Score below 0.1 for 5 consecutive windows, verify demotion to Cold.
  </action>
