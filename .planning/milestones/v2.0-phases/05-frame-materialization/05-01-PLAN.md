<objective>
Implement parked traversers (frames) that materialize multi-hop graph patterns from an anchor node and maintain state through differential operations.

Purpose: Frames are the core value proposition of Krabnet — pre-materialized graph traversal results that are incrementally maintained. A frame holds a multi-hop pattern (Vec<HopSpec>), materializes by DFS from an anchor node, stores complete paths as +1 assertions in a DiffCollection, supports delta application for incremental updates, and can be evicted/re-materialized.

Output: `src/frame.rs` with Frame struct, DFS materialization, delta application, query, snapshot, eviction, re-materialization, and comprehensive tests.
</objective>

<tasks>

<task id="1" name="Create frame module with materialization and operations" commit_prefix="feat(05-01)">
Create `src/frame.rs`:

**Frame struct:**
- `id: u64` — unique frame identifier
- `anchor: NodeId` — root node for materialization
- `pattern: Vec<HopSpec>` — multi-hop traversal pattern
- `state: DiffCollection<Vec<NodeId>>` — materialized paths stored as differential tuples (each path is Vec<NodeId>)
- `tier: FrameTier` — current temperature tier
- `query_count: u64` — number of times this frame has been queried
- `mutation_count: u64` — number of deltas applied
- `last_epoch: Option<Epoch>` — last epoch at which state was modified
- `net_delta: i64` — cached from DiffCollection

**Core operations:**
- `new(id: u64, anchor: NodeId, pattern: Vec<HopSpec>) -> Self` — create empty frame, tier = Cold
- `materialize(&mut self, graph: &Graph, epoch: Epoch)` — DFS from anchor following pattern hops, for each complete path assert_tuple(path, epoch). Each hop: get neighbors from graph filtered by direction and edge_type, then filter by target_type (check node type), then filter by property filter (Filter::PropertyEquals checks property, Filter::HasProperty checks existence, Filter::None passes). Collect all complete paths (length == hops + 1 including anchor).
- `apply_delta(&mut self, path: Vec<NodeId>, epoch: Epoch, delta: Delta)` — apply +1 or -1 to specific path, update mutation_count and last_epoch and net_delta
- `query(&mut self) -> Vec<&Vec<NodeId>>` — return current_state from DiffCollection, increment query_count
- `snapshot(&self, epoch: Epoch) -> Vec<&Vec<NodeId>>` — return snapshot at epoch (does NOT increment query_count)
- `compact(&mut self, frontier: Epoch) -> CompactionResult` — delegate to DiffCollection.compact
- `evict(&mut self)` — clear DiffCollection (replace with new empty), set tier to Cold, reset net_delta to 0
- `rematerialize(&mut self, graph: &Graph, epoch: Epoch)` — evict then materialize

**Helper: DFS materialization logic:**
- Recursive or iterative DFS starting from anchor
- At each hop level, use `graph.neighbors(current_node, hop.direction, hop.edge_type)` to get candidates
- For each candidate, check target_type filter (graph.get_node_type == Some(target_type))
- Check property filter (graph.get_property for PropertyEquals, HasProperty)
- Build path incrementally, when all hops are complete, assert the path

**src/lib.rs:** Add `pub mod frame;` and `pub use frame::Frame;`
