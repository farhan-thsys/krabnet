1. Create `src/mcp.rs` implementing the MCP protocol over stdio:

The MCP protocol uses JSON-RPC 2.0 over stdin/stdout. The server must handle three method types:
- `initialize` - Returns server info and capabilities
- `tools/list` - Returns the list of available tools
- `tools/call` - Executes a specific tool

2. Define the MCP server struct:
```rust
pub struct McpServer {
    engine: Engine,
}
```

Note: MCP server owns the Engine directly (not Arc<RwLock>) because it's single-threaded stdio. No concurrent access.

3. Implement JSON-RPC 2.0 message handling:

Use serde_json for parsing. Define request/response types:

```rust
#[derive(Deserialize)]
struct JsonRpcRequest {
    jsonrpc: String,
    id: Option<serde_json::Value>,
    method: String,
    params: Option<serde_json::Value>,
}

#[derive(Serialize)]
struct JsonRpcResponse {
    jsonrpc: String,
    id: Option<serde_json::Value>,
    #[serde(skip_serializing_if = "Option::is_none")]
    result: Option<serde_json::Value>,
    #[serde(skip_serializing_if = "Option::is_none")]
    error: Option<JsonRpcError>,
}
```

4. Implement the `initialize` handler:
- Return: `{ protocolVersion: "2024-11-05", capabilities: { tools: {} }, serverInfo: { name: "krabnet-mcp", version: "0.1.0" } }`

5. Implement `tools/list` returning 5 tools with their JSON Schema input definitions:

- **krabnet_ingest**: description "Ingest a graph mutation event", input schema with event_type enum and appropriate fields per variant
- **krabnet_register_frame**: description "Register a new frame with anchor and pattern", input schema with anchor_node_id, pattern (array of hop specs), epoch
- **krabnet_query_frame**: description "Query a frame's current materialized paths", input schema with frame_id
- **krabnet_stats**: description "Get engine statistics", no required input
- **krabnet_register_template**: description "Register an embryonic pattern template", input schema with id, pattern, threshold, max_candidates, stale_window

Each tool definition follows the MCP tool schema:
```json
{
  "name": "krabnet_ingest",
  "description": "Ingest a graph mutation event",
  "inputSchema": { "type": "object", "properties": { ... }, "required": [...] }
}
```

6. Implement `tools/call` dispatching to each tool:

For each tool, parse the arguments from `params.arguments`, call the appropriate Engine method, return the result as JSON. Use the same conversion logic pattern as grpc.rs but for serde_json::Value.

- **krabnet_ingest**: Parse event from arguments, call `engine.ingest(event)`, return `{ epoch: N }`
- **krabnet_register_frame**: Parse anchor/pattern/epoch, call `engine.register_frame(...)`, return `{ frame_id: N }`
- **krabnet_query_frame**: Parse frame_id, call `engine.query_frame(frame_id)`, return `{ paths: [[node_ids...], ...] }`
- **krabnet_stats**: Call `engine.stats()`, return serialized EngineStats
- **krabnet_register_template**: Parse template fields, call `engine.register_template(...)`, return `{ registered: true }`

7. Implement the stdio run loop:
```rust
impl McpServer {
    pub fn new(engine: Engine) -> Self { ... }

    pub fn run(&mut self) -> std::io::Result<()> {
        let stdin = std::io::stdin();
        let stdout = std::io::stdout();

        for line in stdin.lock().lines() {
            let line = line?;
            if line.is_empty() { continue; }

            let request: JsonRpcRequest = serde_json::from_str(&line)?;
            let response = self.handle_request(request);

            let out = serde_json::to_string(&response)?;
            writeln!(stdout.lock(), "{}", out)?;
            stdout.lock().flush()?;
        }
        Ok(())
    }
}
```

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_mcp_tools_list() {
        let engine = Engine::new(64);
        let mut server = McpServer::new(engine);

        // Simulate tools/list request
        let request = JsonRpcRequest {
            jsonrpc: "2.0".to_string(),
            id: Some(serde_json::json!(1)),
            method: "tools/list".to_string(),
            params: None,
        };

        let response = server.handle_request(request);
        let result = response.result.unwrap();
        let tools = result["tools"].as_array().unwrap();
        assert_eq!(tools.len(), 5);

        let names: Vec<&str> = tools.iter()
            .map(|t| t["name"].as_str().unwrap())
            .collect();
        assert!(names.contains(&"krabnet_ingest"));
        assert!(names.contains(&"krabnet_register_frame"));
        assert!(names.contains(&"krabnet_query_frame"));
        assert!(names.contains(&"krabnet_stats"));
        assert!(names.contains(&"krabnet_register_template"));
    }
}
```

1. Create `src/bin/krabnet-mcp.rs`:

```rust
//! krabnet-mcp: MCP stdio server for the Krabnet engine.
//!
//! Starts an MCP JSON-RPC 2.0 server reading from stdin and writing to stdout.
//! AI agents connect to this binary via stdio transport.

use krabnet::engine::Engine;
use krabnet::mcp::McpServer;

fn main() {
    // Create engine with default configuration (64-slot ring buffer, no hardening)
    // Production configuration would be larger, but for MCP single-user usage this is fine
    let engine = Engine::new(1024);

    let mut server = McpServer::new(engine);

    if let Err(e) = server.run() {
        eprintln!("krabnet-mcp error: {}", e);
        std::process::exit(1);
    }
}
```
