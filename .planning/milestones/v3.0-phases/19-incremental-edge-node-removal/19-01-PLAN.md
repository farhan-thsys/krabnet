3. **`pub fn retract_edge_removed(pattern: &[HopSpec], graph: &Graph, current_paths: &[&Vec<NodeId>], source: NodeId, target: NodeId) -> EdgeRemovedDeltas`**

   Algorithm:
   - Early return if `pattern.is_empty()` or `current_paths.is_empty()`.
   - For each path in `current_paths`, call `path_broken_by_edge_removal()`.
   - Collect broken paths, deduplicate via `HashSet<Vec<NodeId>>` using `.retain(|p| seen.insert(p.clone()))`.
   - Return `EdgeRemovedDeltas { retracted_paths }`.

   Note: `anchor` is NOT needed as a parameter. The paths already start from the anchor. We only need `pattern` (for direction/edge_type at each hop), `graph` (for parallel edge check), and `source`/`target` of the removed edge.

4. **`fn path_broken_by_edge_removal(path: &[NodeId], pattern: &[HopSpec], graph: &Graph, removed_source: NodeId, removed_target: NodeId) -> bool`** (private helper)

   Algorithm:
   - Guard: if `path.len() != pattern.len() + 1`, return false (invalid path length).
   - For each `(hop_idx, hop)` in `pattern.iter().enumerate()`:
     - Let `from = path[hop_idx]`, `to = path[hop_idx + 1]`.
     - Check if the removed edge matches this hop based on direction:
       - `Direction::Outgoing`: `from == removed_source && to == removed_target`
       - `Direction::Incoming`: `from == removed_target && to == removed_source`
       - `Direction::Any`: either of the above
     - If matched, perform parallel edge survival check: call `graph.neighbors(from, hop.direction, hop.edge_type)` and check if any remaining neighbor `(_, n)` has `n == to`. If no remaining neighbor connects, return `true` (path is broken). If a parallel edge survives, continue checking remaining hops.
   - Return `false` (no hop was broken).

   CRITICAL: The `graph.neighbors()` call uses `hop.direction` and `hop.edge_type` -- this implicitly validates edge type even though `Event::EdgeRemoved` does not carry `type_id`. This is correct because `neighbors()` filters by the hop's type constraint on the post-removal graph state.

5. **`pub fn retract_node_removed(current_paths: &[&Vec<NodeId>], removed_node: NodeId) -> NodeRemovedDeltas`**

   Algorithm:
   - Filter `current_paths` for paths where `path.contains(&removed_node)`.
   - Clone matched paths into `retracted_paths`.
   - No deduplication needed: each path reference in `current_paths` is unique (from `snapshot()` which returns `&Vec<NodeId>` refs).
   - Return `NodeRemovedDeltas { retracted_paths }`.

Update the module-level doc comment at the top of `path_extender.rs` to mention the new retraction functions alongside the existing edge-added extension.

Add `pub use path_extender::{retract_edge_removed, retract_node_removed, EdgeRemovedDeltas, NodeRemovedDeltas}` to `src/lib.rs` re-exports (alongside existing `extend_edge_added` and `EdgeAddedDeltas`).

Do NOT modify `Frame::dfs_collect`, `Frame::materialize`, or `Frame::rematerialize` -- these remain the correctness baseline.
