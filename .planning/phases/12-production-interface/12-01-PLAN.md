2. Create `proto/krabnet.proto` with the following schema:

```protobuf
syntax = "proto3";
package krabnet;

service KrabnetService {
  rpc IngestEvent(IngestEventRequest) returns (IngestEventResponse);
  rpc RegisterFrame(RegisterFrameRequest) returns (RegisterFrameResponse);
  rpc QueryFrame(QueryFrameRequest) returns (QueryFrameResponse);
  rpc SubscribeFrame(SubscribeFrameRequest) returns (stream FrameUpdate);
  rpc ListFrames(ListFramesRequest) returns (ListFramesResponse);
  rpc EvictFrame(EvictFrameRequest) returns (EvictFrameResponse);
  rpc RegisterEmbryonicTemplate(RegisterTemplateRequest) returns (RegisterTemplateResponse);
  rpc GetStats(GetStatsRequest) returns (GetStatsResponse);
}
```

Define message types matching Krabnet's domain types:
- IngestEventRequest: oneof event_type { NodeAdded, NodeRemoved, EdgeAdded, EdgeRemoved, PropertyChanged } with appropriate fields (node_id as uint64, edge_id as uint64, type_id as uint32, etc.)
- IngestEventResponse: uint64 epoch
- RegisterFrameRequest: uint64 anchor_node_id, repeated HopSpec pattern, uint64 epoch
- HopSpec message: Direction enum (OUTGOING/INCOMING/ANY), optional uint32 edge_type, optional uint32 target_type, Filter message
- QueryFrameResponse: repeated Path paths (each Path = repeated uint64 node_ids)
- SubscribeFrameRequest: uint64 frame_id
- FrameUpdate: uint64 frame_id, repeated Path paths, uint64 epoch
- ListFramesResponse: repeated FrameInfo (id, anchor, tier, tuple_count)
- EvictFrameRequest/Response: uint64 frame_id, bool success
- RegisterTemplateRequest: uint64 id, repeated HopSpec pattern, double threshold, uint32 max_candidates, uint64 stale_window
- GetStatsResponse: mirrors EngineStats fields

3. Create `build.rs`:

```rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    tonic_build::compile_protos("proto/krabnet.proto")?;
    Ok(())

//! gRPC server for the Krabnet engine.
//!
//! Implements [`KrabnetService`] with the engine held via `Arc<RwLock<Engine>>`.
//! SubscribeFrame uses `tokio::sync::broadcast` for real-time streaming.

use std::sync::{Arc, RwLock};
use tokio::sync::broadcast;
use tonic::{Request, Response, Status};

// Include generated protobuf types
pub mod proto {
    tonic::include_proto!("krabnet");
}

use proto::krabnet_service_server::{KrabnetService, KrabnetServiceServer};
use proto::*;

use crate::engine::Engine;
use crate::types::*;
// ... other imports as needed
```

2. Define `KrabnetServer` struct:
```rust
pub struct KrabnetServer {
    engine: Arc<RwLock<Engine>>,
    frame_tx: broadcast::Sender<proto::FrameUpdate>,
}
```

The broadcast::Sender capacity should be 1024 (reasonable buffer for frame updates).

3. Implement all 8 RPC methods on KrabnetServer:

- **IngestEvent**: Acquire write lock on engine, convert proto event to krabnet Event, call `engine.ingest(event)`, return epoch. After ingest, if any frame was affected, broadcast FrameUpdate on the channel.
- **RegisterFrame**: Acquire write lock, convert HopSpec from proto, call `engine.register_frame(anchor, pattern, epoch)`, return frame_id.
- **QueryFrame**: Acquire write lock (query_frame takes &mut self), call `engine.query_frame(frame_id)`, convert Vec<Vec<NodeId>> to proto Path messages.
- **SubscribeFrame**: Subscribe to broadcast channel via `frame_tx.subscribe()`. Return a `tokio_stream::wrappers::BroadcastStream` filtered by requested frame_id. Use `ReceiverStream` or manual stream implementation.
- **ListFrames**: Acquire read lock, iterate `engine.stats()` (for counts), but also need frame details. Add a `list_frames()` method to Engine that returns Vec of (frame_id, anchor, tier, tuple_count) if it doesn't exist. Otherwise, iterate internal state. Since Engine fields are private, add a `pub fn list_frames(&self) -> Vec<(u64, NodeId, FrameTier, usize)>` method to Engine.
- **EvictFrame**: Acquire write lock, remove frame from engine. Need to add `pub fn evict_frame(&mut self, frame_id: u64) -> bool` to Engine if not present. Evict should unregister from index and remove from frames map.
- **RegisterEmbryonicTemplate**: Acquire write lock, convert proto to PatternTemplate, call `engine.register_template(template)`.
- **GetStats**: Acquire read lock, call `engine.stats()`, convert EngineStats to proto response.

4. Add conversion functions between proto types and krabnet types:
- `proto_event_to_event(req: &IngestEventRequest) -> Result<Event, Status>`
- `hopspecs_from_proto(hops: &[proto::HopSpec]) -> Result<Vec<HopSpec>, Status>`
- `paths_to_proto(paths: &[Vec<NodeId>]) -> Vec<proto::Path>`

5. Add a public constructor:
```rust
impl KrabnetServer {
    pub fn new(engine: Arc<RwLock<Engine>>) -> Self { ... }
    pub fn into_service(self) -> KrabnetServiceServer<Self> { ... }
}
```

6. Update `src/lib.rs`: add `pub mod grpc;` and `pub use grpc::KrabnetServer;`

7. Add Engine helper methods if needed (add to src/engine.rs):

- `pub fn evict_frame(&mut self, frame_id: u64) -> bool` - unregisters from index, removes from frames map

IMPORTANT: Engine currently uses `&mut self` for query_frame. The gRPC server holds Arc<RwLock<Engine>>, so write lock is needed for query_frame. This is fine.
