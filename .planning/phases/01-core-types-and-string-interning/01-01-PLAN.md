<task type="auto">
  <name>Task 2: Create string interner module with comprehensive tests</name>
  <files>src/interner.rs</files>
  <action>
Create the string interner module following the matklad pattern (HashMap + Vec).

**src/interner.rs:**
- Module-level doc comment explaining the interner's purpose: converts human-readable strings to compact u32 IDs at initialization time. Zero String allocation on hot path after init.
- Include a doc example in the module comment showing basic usage (intern, resolve, idempotency)

Define `pub struct Interner` with:
- `map: HashMap<String, u32>` -- forward lookup (string -> ID)
- `strings: Vec<String>` -- reverse lookup (ID -> string, index is the ID)

Implement methods:
- `pub fn new() -> Self` -- empty interner
- `pub fn with_capacity(capacity: usize) -> Self` -- pre-allocate for known string count
- `pub fn intern(&mut self, s: &str) -> u32` -- intern a string, return stable ID. Idempotent (same string = same ID). MUST assert `self.strings.len() < u32::MAX as usize` before casting to prevent silent overflow. The `&mut self` signature enforces the init-only invariant: once shared as `&Interner`, no new strings can be added.
- `pub fn resolve(&self, id: u32) -> Option<&str>` -- reverse lookup by index into Vec. Returns None for unknown IDs.
- `pub fn len(&self) -> usize` -- number of unique interned strings
- `pub fn is_empty(&self) -> bool` -- whether interner has no strings

Implement `Default for Interner` delegating to `Self::new()`.

Add doc comments on every public method with usage examples and panic conditions.

Add `#[cfg(test)] mod tests` inline with comprehensive tests:
- `intern_returns_stable_id`: intern same string twice, get same ID
- `different_strings_get_different_ids`: intern two different strings, IDs differ
- `resolve_returns_original_string`: intern then resolve returns original
- `resolve_unknown_id_returns_none`: resolve on empty interner returns None
- `ids_are_sequential_from_zero`: first intern = 0, second = 1, third = 2
- `len_tracks_unique_strings`: len increments only for new strings, not duplicates
- `with_capacity_does_not_affect_behavior`: same behavior as new()
- `all_interned_strings_resolvable`: intern 5 strings, resolve all successfully
- `empty_string_can_be_interned`: empty string "" is valid input
- `is_empty_reflects_state`: empty after new, not empty after intern
- `default_creates_empty`: Default::default() == new()
