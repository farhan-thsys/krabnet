Three changes to `src/engine.rs`:

**Change 1: DeletionContext struct**

Add a simple struct above or near the `Engine` struct definition:

```rust
/// Context captured before a node is removed from the graph.
///
/// Stored temporarily in `ingest()` Step 2 before `graph.remove_node()`
/// destroys the node's adjacency. Currently holds only the node_id --
/// the retraction algorithm scans paths for node presence rather than
/// using edge adjacency.
#[derive(Debug, Clone)]
pub(crate) struct DeletionContext {
    /// The node being removed.
    pub node_id: NodeId,
}
```

**Change 2: Modify `maintain_and_evaluate_frames` signature and dispatch**

Add a `force_rematerialize: bool` parameter to the function signature:

```rust
fn maintain_and_evaluate_frames(
    frames: &[(u64, Arc<RwLock<Frame>>)],
    graph: &Graph,
    epoch: Epoch,
    prev_deltas: &HashMap<u64, i64>,
    event: &Event,
    force_rematerialize: bool,  // NEW
) -> Vec<(u64, i64)> {
```

Inside the scoped thread, update the `match event` block:

```rust
if force_rematerialize {
    frame.rematerialize(graph, epoch);
} else {
    match event {
        Event::EdgeAdded { source, target, type_id, .. } => {
            // Existing incremental +1 path extension (unchanged)
            let deltas = crate::path_extender::extend_edge_added(
                frame.anchor(), frame.pattern(), graph,
                *source, *target, *type_id,
            );
            for path in deltas.new_paths {
                frame.apply_delta(path, epoch, Delta(1));
            }
        }
        Event::EdgeRemoved { source, target, .. } => {
            // NEW: Incremental -1 retraction
            let current = frame.snapshot(Epoch(u64::MAX));
            let deltas = crate::path_extender::retract_edge_removed(
                frame.pattern(), graph,
                &current, *source, *target,
            );
            for path in deltas.retracted_paths {
                frame.apply_delta(path, epoch, Delta(-1));
            }
        }
        Event::NodeRemoved { node_id } => {
            // NEW: Incremental -1 retraction (node presence scan)
            let current = frame.snapshot(Epoch(u64::MAX));
            let deltas = crate::path_extender::retract_node_removed(
                &current, *node_id,
            );
            for path in deltas.retracted_paths {
                frame.apply_delta(path, epoch, Delta(-1));
            }
        }
        _ => {
            // Fallback: full re-traverse for PropertyChanged and NodeAdded
            frame.rematerialize(graph, epoch);
        }
    }
}
```

Update the function's doc comment to document the new `force_rematerialize` parameter.

Add new oracle tests to the `#[cfg(test)]` module in `src/engine.rs`, near the existing oracle tests (after test 9). Use the existing `oracle_check()` helper function pattern.

Each oracle test must: (1) set up graph, (2) register frame, (3) ingest events including removal, (4) call `oracle_check()` to verify incremental result matches full re-traverse.

**New oracle tests:**

1. **`test_oracle_multi_hop_edge_removed_middle`** (Test 10)
   - Set up: 3 nodes (1, 2, 3), edges 1->2 and 2->3 (both TypeId(100)).
   - Register 2-hop frame anchored at NodeId(1) with pattern [Outgoing/TypeId(100), Outgoing/TypeId(100)].
   - Ingest: Add nodes, add edges, then EdgeRemoved for edge 2->3.
   - Oracle check: frame should have 0 paths (the only path [1,2,3] is broken at hop 1).

2. **`test_oracle_parallel_edge_removal_survives`** (Test 11)
   - Set up: 2 nodes (1, 2), TWO outgoing edges 1->2 (both TypeId(100), different EdgeIds).
   - Register 1-hop frame anchored at NodeId(1) with pattern [Outgoing/TypeId(100)].
   - Ingest: Add nodes, add both edges, then EdgeRemoved for the first edge.
   - Oracle check: frame should STILL have path [1, 2] (second parallel edge survives). This validates the parallel edge check in retract_edge_removed.

3. **`test_oracle_node_removed_diamond`** (Test 12)
   - Set up: 4 nodes forming a diamond: 1->2, 1->3, 2->4, 3->4. All TypeId(100).
   - Register 2-hop frame anchored at NodeId(1): [Outgoing/TypeId(100), Outgoing/TypeId(100)].
   - After edges: two paths exist: [1,2,4] and [1,3,4].
   - Ingest: NodeRemoved for node 2.
   - Oracle check: only path [1,3,4] should remain ([1,2,4] retracted because node 2 is gone).

4. **`test_oracle_multi_frame_edge_removal`** (Test 13)
   - Set up: 3 nodes (1, 2, 3), edges 1->2 and 2->3.
   - Register TWO frames: Frame A anchored at NodeId(1) 1-hop [Outgoing/TypeId(100)], Frame B anchored at NodeId(1) 2-hop [Outgoing/TypeId(100), Outgoing/TypeId(100)].
   - Ingest: EdgeRemoved for edge 1->2.
   - Oracle check both frames: Frame A loses [1,2], Frame B loses [1,2,3]. Both frames updated by the same event.

5. **`test_oracle_sequential_add_remove_add_remove`** (Test 14)
   - Set up: 2 nodes (1, 2).
   - Register 1-hop frame anchored at NodeId(1) [Outgoing/TypeId(100)].
   - Sequence: EdgeAdded 1->2 (path appears), EdgeRemoved 1->2 (path retracted), EdgeAdded 1->2 again (path re-appears), EdgeRemoved 1->2 again (path retracted again).
   - Oracle check after each step: 1 path, 0 paths, 1 path, 0 paths.

6. **`test_oracle_node_removed_cascade_no_ghost_paths`** (Test 15)
   - Set up: Chain 1->2->3->4, all TypeId(100).
   - Register 3-hop frame anchored at NodeId(1).
   - Ingest: NodeRemoved for node 3.
   - Oracle check: path [1,2,3,4] is retracted (node 3 at position 2 is gone). No ghost paths remain.

Use `oracle_check()` after each relevant ingest step. Assert the oracle returns true (incremental state matches fresh materialization).
