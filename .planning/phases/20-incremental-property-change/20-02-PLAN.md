In `maintain_and_evaluate_frames` (engine.rs, currently around line 826), find the event dispatch `match` block. Currently there is:
- `Event::EdgeAdded { .. }` -> incremental +1 via extend_edge_added (Phase 18)
- `Event::EdgeRemoved { .. }` -> incremental -1 via retract_edge_removed (Phase 19)
- `Event::NodeRemoved { .. }` -> incremental -1 via retract_node_removed (Phase 19)
- `_ =>` -> full rematerialize fallback (handles PropertyChanged and NodeAdded)

Replace the `_ =>` catch-all with an explicit `Event::PropertyChanged` arm and a narrower `_ =>` that only handles NodeAdded:

```rust
Event::PropertyChanged { node_id, .. } => {
    let current = frame.snapshot(Epoch(u64::MAX));
    let current_refs: Vec<&Vec<NodeId>> = current.iter().collect();
    let deltas = crate::path_extender::reevaluate_property_changed(
        frame.anchor(),
        frame.pattern(),
        graph,
        &current_refs,
        *node_id,
    );
    for path in deltas.retracted_paths {
        frame.apply_delta(path, epoch, Delta(-1));
    }
    for path in deltas.new_paths {
        frame.apply_delta(path, epoch, Delta(1));
    }
}
_ => {
    // Only NodeAdded remains -- nodes alone cannot create paths
    frame.rematerialize(graph, epoch);
}
```

IMPORTANT: The `current_paths` parameter of `reevaluate_property_changed` expects `&[&Vec<NodeId>]`. Use `frame.snapshot(Epoch(u64::MAX))` to get the current materialized paths, then convert to references. Match the exact pattern used by retract_edge_removed in the EdgeRemoved arm (examine how `current` is obtained and passed there).

IMPORTANT: The `force_rematerialize` check MUST remain above the match -- if force_rematerialize is true, the match is bypassed entirely (this is the coalescer path from Phase 19).

Also update the comment on the `_ =>` arm to note that only `NodeAdded` remains and that all other event types are handled incrementally.
