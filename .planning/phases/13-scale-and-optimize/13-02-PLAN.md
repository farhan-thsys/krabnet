Create `src/trunk.rs` implementing trunk/leaf detection (TRUNK-01, TRUNK-02):

**Core concept:** A "trunk" is a sub-path (sequence of HopSpecs) that appears as a prefix or contiguous sub-sequence in multiple frame patterns. Frames sharing trunk paths benefit from Hot pinning since they're structural spines accessed by many queries.

**Data structures:**
```rust
/// Result of trunk detection: which sub-paths are trunks and which frames they span.
#[derive(Debug, Clone)]
pub struct TrunkInfo {
    /// Sub-path that qualifies as a trunk.
    pub path: Vec<HopSpec>,
    /// Frame IDs sharing this trunk.
    pub frame_ids: Vec<u64>,
}
```

**Public API (TRUNK-01):**
- `pub fn detect_trunks(frames: &[(u64, Vec<HopSpec>)], min_shared_count: usize) -> Vec<TrunkInfo>`
  - Takes (frame_id, pattern) pairs and a minimum sharing threshold.
  - For each frame pattern, generate all contiguous sub-paths of length >= 1 (use sliding window).
  - Count how many frames share each sub-path (use `HashMap<Vec<HopSpec>, Vec<u64>>`).
  - Return sub-paths with `frame_ids.len() >= min_shared_count` as TrunkInfo.
  - Sort results by frame count descending (most shared first).

- `pub fn classify_frames(frames: &[(u64, Vec<HopSpec>)], trunk_infos: &[TrunkInfo]) -> (HashSet<u64>, HashSet<u64>)`
  - Returns `(trunk_frame_ids, leaf_frame_ids)`.
  - A frame is a "trunk frame" if its pattern appears as a trunk sub-path (or contains one).
  - A frame is a "leaf frame" if it does NOT appear in any trunk.
  - All frames in any TrunkInfo.frame_ids are trunk frames; the rest are leaf frames.

**TRUNK-02 — Hot pinning:**
- `pub fn pinned_frame_ids(trunk_infos: &[TrunkInfo]) -> HashSet<u64>`
  - Returns the set of all frame IDs that participate in any trunk (these should be pinned to Hot).

**Implementation notes:**
- HopSpec must implement PartialEq + Eq + Hash for use as HashMap key. Check if it already does -- if not, derive these traits on HopSpec in types.rs (it currently derives Debug, Clone, Copy, PartialEq, Eq, Hash -- verify). Actually HopSpec has Option fields and Filter enum -- verify all are hashable. If Filter doesn't implement Hash, add it.
- Actually, HopSpec contains `filter: Filter` and Filter contains `PropertyValue` which has `f64` (Float variant) -- this means we CANNOT derive Hash on HopSpec directly. Instead, serialize the sub-path to a canonical string representation for HashMap keying:
  - `fn hop_key(hop: &HopSpec) -> String` — format as "{direction:?}|{edge_type:?}|{target_type:?}|{filter:?}"
  - `fn path_key(path: &[HopSpec]) -> String` — join hop_keys with "/"
  - Use `HashMap<String, Vec<u64>>` for counting.

Create `src/buffer_pool.rs` implementing a custom buffer pool (BUFPOOL-01, BUFPOOL-02):

**BufferPool struct (BUFPOOL-01):**
```rust
pub struct BufferPool {
    /// Contiguous backing buffer.
    buffer: Vec<u8>,
    /// Size of each page in bytes.
    page_size: usize,
    /// Total number of pages.
    page_count: usize,
    /// Free list: indices of available pages.
    free_list: Vec<usize>,
    /// Allocated pages: maps page_index -> PageMeta.
    allocated: HashMap<usize, PageMeta>,
}

#[derive(Debug, Clone)]
pub struct PageMeta {
    /// Which frame owns this page (if any).
    pub frame_id: Option<u64>,
    /// Tier of the owning frame at allocation time.
    pub tier: FrameTier,
}

/// Handle returned by alloc, used for read/write/free.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct PageHandle(pub usize);
```

**Public API:**
- `pub fn new(total_bytes: usize, page_size: usize) -> Self`
  - Default: 256 MB total, 4 KB pages = 65,536 pages.
  - Allocate `vec![0u8; total_bytes]`, compute page_count, populate free_list with 0..page_count.
  - For testing, accept smaller sizes.

- `pub fn alloc(&mut self, meta: PageMeta) -> Option<PageHandle>`
  - Pop from free_list, store meta in allocated, return PageHandle.
  - Returns None if no free pages.

- `pub fn free(&mut self, handle: PageHandle)`
  - Remove from allocated, push index back to free_list.
  - Panics if handle not allocated (debug assert).

- `pub fn read(&self, handle: PageHandle, offset: usize, len: usize) -> &[u8]`
  - Returns slice from `buffer[page_start + offset .. page_start + offset + len]`.
  - Panics if offset+len > page_size or handle not allocated.

- `pub fn write(&mut self, handle: PageHandle, offset: usize, data: &[u8])`
  - Copies data into `buffer[page_start + offset ..]`.
  - Panics if offset+data.len() > page_size or handle not allocated.

- `pub fn free_page_count(&self) -> usize` — returns free_list.len()
- `pub fn allocated_page_count(&self) -> usize` — returns allocated.len()
- `pub fn total_page_count(&self) -> usize` — returns page_count

**Graph-aware eviction (BUFPOOL-02):**
- `pub fn evict_by_tier(&mut self, tier: FrameTier) -> Vec<PageHandle>`
  - Find all allocated pages with matching tier, free them, return their handles.
  - Used by engine to free Cold pages first, then Warm, never Hot.

- `pub fn evict_coldest(&mut self, count: usize) -> Vec<PageHandle>`
  - Evict up to `count` pages in priority order: Cold first, then Warm, never Hot.
  - Collect Cold pages, if not enough add Warm pages, never touch Hot.
  - Free each evicted page, return handles.

- `pub fn update_tier(&mut self, handle: PageHandle, new_tier: FrameTier)`
  - Update the tier in the page's metadata (e.g., after hysteresis tier change).

**Implementation notes:**
- Use `HashMap<usize, PageMeta>` for O(1) alloc/free by page index.
- Free list is a Vec used as a stack (push/pop from end) for O(1) alloc.
- Buffer is a single `Vec<u8>` -- page N starts at `N * page_size`.
- For testing, use small sizes (e.g., 4096 total bytes, 256 byte pages = 16 pages).

**Tests (co-located in buffer_pool.rs):**

- `test_read_write_roundtrip` — write bytes, read back, verify match.
- `test_update_tier` — allocate as Cold, update to Hot, verify evict_coldest skips it.
