**Update PatternTemplate (LEARN-01):**

Add success/failure tracking fields to `PatternTemplate`:
```rust
pub struct PatternTemplate {
    pub id: u64,
    pub pattern: Vec<HopSpec>,
    pub threshold: f64,
    pub max_candidates: usize,
    pub stale_window: u64,
    /// Number of successful promotions from this template (LEARN-01).
    pub success_count: u64,
    /// Number of failed candidates (pruned/capped without promotion) from this template (LEARN-01).
    pub failure_count: u64,
    /// Whether this template is active (LEARN-02). Deactivated templates skip observation.
    pub active: bool,
}
```

Update existing `template()` helper in tests and all existing PatternTemplate construction sites to include `success_count: 0, failure_count: 0, active: true`.

**Update EmbryonicDiscovery:**

1. **Track success on promotion:** In `observe_edge`, when a candidate is promoted, increment the template's `success_count`.

2. **Track failure on prune/cap:** In `prune_stale`, when a candidate is removed as stale, increment the template's `failure_count`. In `enforce_caps`, when excess candidates are removed, increment `failure_count` by the number removed.

3. **Template sorting for observe_edge scanning (LEARN-02):**
   - At the start of `observe_edge`, sort `template_ids` by `success_ratio()` descending so higher-success templates are scanned first.
   - Add method to PatternTemplate: `pub fn success_ratio(&self) -> f64` — returns `success_count as f64 / (success_count + failure_count).max(1) as f64`.

4. **Deactivation (LEARN-02):**
   - After incrementing failure_count in prune_stale/enforce_caps, check if `success_count + failure_count >= 50` AND `success_ratio() < 0.1`. If so, set `active: false`.
   - In `observe_edge`, skip templates where `active == false`.
   - Add `pub fn deactivated_template_count(&self) -> usize` — count templates with active=false.

5. **Public methods:**
   - `pub fn template_success_ratio(&self, template_id: u64) -> Option<f64>` — returns success_ratio for a template.
   - `pub fn reactivate_template(&mut self, template_id: u64)` — sets active=true (for manual override).

**Tests (co-located in embryonic.rs, TEST-31):**
- `test_learned_weighting` (TEST-31): Register 2 templates. Feed edges that complete template A's pattern 10 times (success_count=10) and never complete template B (prune B's candidates as stale 10 times, failure_count=10). Verify template A's success_ratio > template B's success_ratio.
- `test_deactivation_after_50` — Register template, generate 50 stale-pruned candidates (failure_count=50, success_count=0). Verify template is deactivated (active=false). Verify subsequent observe_edge skips it.
- `test_success_increments_on_promotion` — Promote a candidate, verify success_count incremented.
- `test_failure_increments_on_prune` — Prune stale candidates, verify failure_count incremented.

Ensure ALL existing embryonic.rs tests still pass. The new fields have defaults (0, 0, true) that preserve backward compatibility.
