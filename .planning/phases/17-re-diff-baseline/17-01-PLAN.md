<task type="auto">
  <name>Task 2: Build correctness oracle test harness with multi-mutation scenarios</name>
  <files>src/engine.rs</files>
  <action>
Add a `#[cfg(test)]` oracle test module at the bottom of engine.rs (inside the existing tests module or as a new submodule). This harness is the verification backbone for all future incremental work (Phases 18-20).

1. Create an `oracle_check` helper function:
   ```rust
   #[cfg(test)]
   fn oracle_check(engine: &mut Engine, frame_id: u64) {
       // Get the maintained frame's current paths
       let maintained_paths: HashSet<Vec<NodeId>> = engine
           .query_frame(frame_id)
           .expect("Frame must exist")
           .into_iter()
           .collect();

       // Get frame metadata to build reference
       let (anchor, pattern) = {
           let frame_arc = engine.frames.get(&frame_id).expect("Frame must exist");
           let frame = frame_arc.read().expect("RwLock poisoned");
           (frame.anchor(), frame.pattern().to_vec())
       };

       // Build fresh reference frame from scratch
       let epoch = engine.current_epoch();
       let mut reference = Frame::new(u64::MAX, anchor, pattern);
       reference.materialize(engine.graph(), epoch);
       let expected_paths: HashSet<Vec<NodeId>> = reference
           .query()
           .into_iter()
           .cloned()
           .collect();

       assert_eq!(
           maintained_paths, expected_paths,
           "Oracle mismatch for frame {}: maintained {} paths, expected {} paths.\nMaintained: {:?}\nExpected: {:?}",
           frame_id, maintained_paths.len(), expected_paths.len(),
           maintained_paths, expected_paths,
       );
   }
   ```

2. Test: `test_oracle_edge_added_after_registration`
   - Create engine, add 3 nodes (types 10, 20, 20), add edge 1->2 (type 100)
   - Register frame: anchor=1, 1-hop outgoing type 100, target_type 20
   - oracle_check (should have 1 path: [1,2])
   - Add edge 1->3 (type 100) via ingest
   - oracle_check (should have 2 paths: [1,2] and [1,3])

3. Test: `test_oracle_edge_removed`
   - Create engine, add 3 nodes, add edges 1->2 and 1->3 (type 100)
   - Register frame: anchor=1, 1-hop outgoing type 100, target_type 20
   - oracle_check (2 paths)
   - Remove edge 1->2 via EdgeRemoved event
   - oracle_check (1 path remaining)
   - Note: Verify EdgeRemoved uses the correct edge_id. The graph auto-assigns edge IDs via next_edge_id counter. The first EdgeAdded gets edge_id from the graph (not the Event). Check if EdgeRemoved needs the graph-assigned ID or the Event-provided ID. Test both paths if needed.

4. Test: `test_oracle_node_removed`

   - Register frame: anchor=1, 1-hop outgoing
   - oracle_check (2 paths)
   - Remove node 2 via NodeRemoved event
   - oracle_check (1 path: [1,3] only)

5. Test: `test_oracle_property_changed`

7. Test: `test_oracle_unaffected_frame_unchanged`

Each test should call oracle_check after every mutation to ensure continuous correctness.

Important: Use the actual Event enum variants from types.rs. Check which events the graph.rs mutation methods handle (NodeRemoved cascading edge removal, PropertyChanged update). Ensure the ingest pipeline routes these events through the inverted index correctly.
