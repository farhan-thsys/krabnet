<task type="auto">
  <name>Task 1: Implement integration tests TEST-09 through TEST-14 and wire coalescer/fanout/hysteresis into engine</name>
  <files>src/engine.rs</files>
  <action>
This task wires the coalescer, fanout limiter, and hysteresis into the Engine (completing the integration that plans 01/02 left as standalone modules), then writes the integration tests.

1. Update Engine struct to include optional coalescer, fanout limiter, and per-frame hysteresis state:
   - Add `coalescer: Option<MutationCoalescer>` field
   - Add `fanout_limiter: Option<FanOutLimiter>` field
   - Add `hysteresis: HashMap<u64, HysteresisState>` field
   - Add a builder-style method or extended constructor: `Engine::with_config(ring_buffer_capacity: usize, compaction_threshold: Option<usize>, coalesce_window: Option<u64>, max_fanout: Option<usize>) -> Self`
   - In `ingest()`, integrate:
     a. If coalescer is Some, push event through coalescer. Only proceed with frame evaluation when coalescer returns a batch (or on flush). For non-coalesced path (coalescer is None), proceed as before.
     b. After getting affected frames from inverted index, if fanout_limiter is Some, run `limit()` on the affected set. Only evaluate the immediate set; deferred frames are queued.
     c. After computing priority scores during evaluation, update hysteresis state per frame. Use hysteresis to determine actual tier changes instead of raw recommend_tier.

2. Add these tests to the existing `#[cfg(test)] mod tests` in engine.rs:

   **test_background_compaction (TEST-09):**
   - Create engine with compaction enabled (threshold: 1000)
   - Ingest 50K events (add nodes and edges in batches)
   - Register frames that will accumulate many tuples
   - Wait briefly for background compaction to fire (std::thread::sleep for 100ms)
   - Verify CompactionStats shows compactions_completed > 0
   - Verify tuple counts decreased
   - Verify query_frame still returns correct results

   **test_concurrent_frame_eval (TEST-10):**
   - Create engine with default config
   - Add 100 nodes and create 100 frames (each anchored at different nodes)
   - Ingest 10K edge events that affect multiple frames
   - Query all 100 frames and verify each has correct state
   - The test validates correctness -- parallelism is internal to ingest()

   **test_coalescing_deduplicates (TEST-11):**
   - Create engine with coalescer (window_size: 100)
   - Register a frame anchored at node 1
   - Ingest 100 PropertyChanged events all targeting node 1 within the window
   - Count how many times the frame was actually evaluated (add a counter or check mutation_count)
   - Verify frame evaluation triggered once (coalesced), not 100 times

   **test_coalescing_preserves_different_nodes (TEST-12):**
   - Create engine with coalescer (window_size: 100)
   - Register 10 frames, each anchored at different nodes
   - Ingest mutations to 10 different nodes
   - Flush coalescer
   - Verify all 10 nodes produced triggers (batch has 10 entries)

   **test_fanout_limit (TEST-13):**
   - Create engine with fanout limiter (max_fanout: 1000)
   - Create 2000 frames all registered under the same node (super-node)
   - Ingest an event on that node
   - Verify only MAX_FANOUT (1000) frames were evaluated immediately
   - Verify remainder are in the deferred queue

   **test_hysteresis_prevents_thrashing (TEST-14):**
   - Create a HysteresisState with required_consecutive=5
   - Start frame at Warm tier
   - Alternate scores: 0.1, 0.8, 0.1, 0.8... for 20 iterations
   - Verify frame stays Warm throughout (never reaches 5 consecutive below/above)

3. Ensure all existing engine tests still pass after the Engine struct changes. The key is that Engine::new() continues to work without coalescer/fanout/hysteresis (all None), so existing tests are unaffected.

1. Add these stress/integration tests to `#[cfg(test)] mod tests` in engine.rs:

   **test_sustained_throughput (TEST-15):**
   - Create engine with compaction enabled (threshold: 5000)
   - Build initial graph: 10K nodes, 100K edges (chain + cross-links pattern from benchmarks), 200 frames
   - Record start time and initial memory (use engine.stats().total_tuples as proxy for memory)
   - Ingest 500K events (mix of EdgeAdded and PropertyChanged) in a tight loop
   - Record elapsed time
   - Calculate events/sec = 500_000 / elapsed_secs
   - Assert events/sec > 50_000 (the target throughput)
   - Check memory stability: final total_tuples should not be monotonically increasing (compaction keeps it bounded). Specifically, verify final total_tuples < 2 * peak_tuples_during_run (not unbounded growth)
   - Note: This test may take 10+ seconds. Mark with `#[ignore]` and document that it should be run with `cargo test -- --ignored` for CI.

   **test_compaction_under_load (TEST-16):**
   - Create engine with compaction enabled (threshold: 1000)
   - Ingest a burst of 10K events rapidly (nodes, edges, properties mixed)
   - Register 50 frames
   - Ingest another 10K events (should trigger compaction)
   - Wait 200ms for compaction worker to process
   - Verify CompactionStats.compactions_completed > 0
   - Query every frame -- no panics, results are valid (not corrupt)

   **test_concurrent_read_write (TEST-17):**
   - Create engine (Engine needs to be wrapped in Arc<Mutex> for sharing across threads)
   - Spawn a writer thread: loops for 5 seconds, ingesting events
   - Spawn a reader thread: loops for 5 seconds, querying random frames
   - Join both threads -- no panics
   - Verify engine state is consistent (stats make sense, frame queries return valid data)
   - Note: Since Engine uses &mut self for ingest and &mut self for query_frame, this test verifies that with proper external locking (Arc<Mutex<Engine>>), the system doesn't deadlock or panic. Mark with `#[ignore]` due to duration.

2. Add bench_concurrent_ingest to `benches/krabnet_bench.rs` (BENCH-02):

     - Measured: Ingest 100 events through the full pipeline
     - Use `iter_batched` with `SmallInput` (same pattern as existing benchmarks)
