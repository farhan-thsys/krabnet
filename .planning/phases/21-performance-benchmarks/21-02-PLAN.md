**Test structure:**

1. **Engine setup:**
   `Engine::with_config(1024, Some(10_000), Some(16), Some(1000))` -- compaction at 10K tuples to reduce compaction frequency during stress, coalescing window 16, fanout limit 1000.

2. **Build initial graph (1K nodes, ~2K edges):**
   - 1000 nodes: `NodeId(1)..NodeId(1000)` with alternating types `TypeId(10 + (i % 3) as u32)` (so TypeId(10), TypeId(11), TypeId(12)).
   - 999 chain edges: `NodeId(i) -> NodeId(i+1)` with `TypeId(100)`, EdgeId starting from 0.
   - Track `edge_id_counter` and maintain a `Vec<(u64, u64, u64)>` of `(edge_id, source_id, target_id)` for active edges (needed for EdgeRemoved).

3. **Register 20 multi-hop frames:**
   - Anchors at `NodeId(1), NodeId(11), NodeId(21), ..., NodeId(191)` (step by 10, 20 frames).
   - 2-hop pattern: `[HopSpec { Outgoing, Some(TypeId(100)), None, Filter::None }, HopSpec { Outgoing, Some(TypeId(100)), None, Filter::None }]`.
   - Register at a safe epoch past all initial ingested events.
   - Collect frame IDs into a `Vec<u64>`.

4. **Stress loop (500K events):**
   Track `remove_idx: usize = 0` for EdgeRemoved consumption from `active_edges`.

   For each event `i` in `0..500_000u64`:
   - `i % 5 == 0 | 1 | 2` (60%): **EdgeAdded** -- source=`NodeId((i % 999) + 1)`, target=`NodeId((i % 999) + 2)`, type=`TypeId(100)`. Push to `active_edges`. Increment `edge_id_counter`.
   - `i % 5 == 3` (20%): **EdgeRemoved** -- pop from `active_edges[remove_idx]` if `remove_idx < active_edges.len()`. Increment `remove_idx`.
   - `i % 5 == 4` (20%): **PropertyChanged** -- `NodeId((i % 999) + 1)`, key=0, value=`PropertyValue::Integer(i as i64)`.

   **Oracle check every 10K events:** `if i % 10_000 == 0 && i > 0 { oracle_check(&mut engine, frame_ids[0]); }`

5. **Throughput assertion:**
   `let elapsed = start.elapsed().as_secs_f64(); let eps = 500_000.0 / elapsed; assert!(eps > 50_000.0, "Expected >50K events/sec, got {eps:.0}");`
   Start the timer AFTER initial graph setup, BEFORE the stress loop.

6. **Final oracle check on ALL frames:**
   `for &fid in &frame_ids { oracle_check(&mut engine, fid); }`

**Key design rules (from research):**
- Do NOT use random IDs -- use deterministic modular arithmetic
- EdgeRemoved must target actually-existing edges (consume from active_edges Vec)
- Skip NodeRemoved in stress test (destroys graph topology, makes oracle unreliable over 500K events)
- Oracle check frequency: every 10K events (50 checks total + final check on all 20 frames)
- Use high compaction threshold (10K) to reduce compaction interference
- Do NOT modify any existing tests or functions
