Add the following new benchmark functions to benches/krabnet_bench.rs, AFTER the existing benchmarks and BEFORE the `criterion_group!` macro. Do NOT modify any existing benchmark functions.

**1. Helper function `setup_scaling_graph(n: u64)`:**
Build a graph with `n` nodes (NodeId(1)..NodeId(n), all TypeId(10)), a chain of edges (1->2->3->...->n, TypeId(100)), and a 1-hop frame at anchor=NodeId(1) with pattern `[HopSpec { direction: Outgoing, edge_type: Some(TypeId(100)), target_type: None, filter: Filter::None }]`. Materialize the frame at Epoch(n+10). Add a NEW node NodeId(n+1) with TypeId(10) and edge NodeId(1)->NodeId(n+1) with TypeId(100). Return `(graph, frame, pattern, NodeId(n+1))`. Assert that the frame has at least 1 materialized path after setup.

**2. `bench_incremental_scaling(c: &mut Criterion)` (PERF-01):**
Use `criterion::BenchmarkGroup` named "incremental_scaling". For each scale in `[100u64, 1_000, 10_000]`:
- Benchmark `extend_edge_added` with `BenchmarkId::new("extend_edge_added", node_count)`, using `iter_batched` with the setup_scaling_graph helper. The measured section calls ONLY `extend_edge_added(frame.anchor(), &pattern, &graph, NodeId(1), new_target, TypeId(100))`.
- Benchmark full rematerialize with `BenchmarkId::new("full_rematerialize", node_count)`, using same setup. The measured section calls ONLY `frame.rematerialize(&graph, Epoch(n+20))`.
- Use `BatchSize::SmallInput` for 100/1000, `BatchSize::LargeInput` for 10000.
Call `group.finish()` at the end.

**3. `bench_incremental_edge_added(c: &mut Criterion)` (PERF-02):**
Setup: Build a graph with 50 nodes (TypeId(10)), chain 1->2->...->50 (TypeId(100)), and cross-links every 5th node (TypeId(200)). Register a 2-hop frame at anchor=NodeId(1) with pattern `[HopSpec { Outgoing, Some(TypeId(100)), None, None }, HopSpec { Outgoing, Some(TypeId(100)), None, None }]`. Materialize at Epoch(100). Then add edge NodeId(2)->NodeId(40) with TypeId(100). Assert frame has paths. Measured: `extend_edge_added(frame.anchor(), &pattern, &graph, NodeId(2), NodeId(40), TypeId(100))`.

**4. `bench_rematerialize_edge_added(c: &mut Criterion)` (PERF-02 baseline):**
Identical setup to bench_incremental_edge_added. Measured: `frame.rematerialize(&graph, Epoch(200))`.

**5. `bench_incremental_edge_removed(c: &mut Criterion)` (PERF-03):**
Setup: Build same 50-node graph as PERF-02. Register 2-hop frame, materialize. The edge to remove is the chain edge from NodeId(2)->NodeId(3) (EdgeId for i=1 in the chain). Remove it via `graph.remove_edge(NodeId(2), NodeId(3))`. Take `snapshot(Epoch(u64::MAX))`, convert to `Vec<&Vec<NodeId>>`. Assert frame has paths. Measured: `retract_edge_removed(&pattern, &graph, &current_refs, NodeId(2), NodeId(3))`.

**6. `bench_rematerialize_edge_removed(c: &mut Criterion)` (PERF-03 baseline):**
Identical setup to bench_incremental_edge_removed. Measured: `frame.rematerialize(&graph, Epoch(200))`.

**Update `criterion_group!`:** Add all 5 new benchmark functions to the existing `criterion_group!(benches, ...)` macro call.

**Import additions:** Add `use criterion::BenchmarkId;` and `use krabnet::path_extender::{extend_edge_added, retract_edge_removed};` if not already imported via `use krabnet::*`. **Key design rules (from research):**
- Use `iter_batched` to separate setup from measurement -- never measure graph construction
- Use deterministic node IDs (no randomness)
- Use `black_box()` around measured function returns
- Do NOT modify existing benchmarks
- For edge removal, the graph mutation (remove_edge) goes in SETUP, not measured section
- For edge addition, the graph mutation (add_edge/add_node) goes in SETUP, not measured section
